# Event Manager API

## 📌 Опис проєкту

Event Manager API — це RESTful backend-додаток для управління подіями (events), який реалізує повний CRUD-функціонал, а
також роботу із зображеннями (upload, зберігання, отримання тимчасових URL).

Проєкт побудований з урахуванням production-підходів:

- чітке розділення шарів (Controller / Service / Repository)
- dependency injection
- централізована обробка помилок
- валідація вхідних даних
- логування
- масштабованість та розширюваність

---

## 🏗️ Архітектура

Проєкт реалізовано за багатошаровою архітектурою:

- **Controller** — обробка HTTP-запитів
- **Service** — бізнес-логіка
- **Repository** — робота з базою даних
- **Middleware** — валідація, логування, обробка помилок
- **Utils** — допоміжні функції

Використовується підхід **Dependency Injection**, що дозволяє легко тестувати та розширювати систему.

---

## 🚀 Основний функціонал

### 📅 Events

- Створення події
- Отримання списку подій (з пагінацією, сортуванням)
- Отримання події за ID
- Оновлення події
- Видалення події

### 🖼 Робота з файлами

- Завантаження зображень (до 5 файлів)
- Збереження метаданих у БД
- Зберігання файлів у S3-сумісному сховищі (MinIO)
- Генерація тимчасових URL для доступу

### 📊 Додатково

- Пагінація (`limit`, `page`)
- Сортування (`sortBy`, `order`)
- Обробка помилок з єдиним форматом відповіді
- Логування HTTP-запитів

---

## 🗄️ База даних

Використовується PostgreSQL.

### Основні таблиці:

- `events` — події
- `files` — метадані файлів

📌 У проєкті також використовується діаграма зв'язків таблиць (ER Diagram).

## 🧰 Технології та бібліотеки

### Backend

- **express** — веб-фреймворк
- **pg** — робота з PostgreSQL
- **zod** — валідація даних
- **multer** — обробка file upload
- **uuid** — генерація унікальних ключів

### Storage

- **@aws-sdk/client-s3** — робота з S3 API
- **@aws-sdk/s3-request-presigner** — генерація signed URL
- **MinIO** — S3-сумісне локальне сховище

### Логування

- **pino**
- **pino-http**
- **pino-pretty** (dev)

### Інфраструктура

- **Docker**
- **Docker Compose**
- **Redis** — кешування (готово до використання)

### Тестування

- **jest**

## 🐳 Інфраструктура (Docker Compose)

Проєкт піднімається через Docker Compose та включає:

- `app` — Node.js сервер
- `postgres` — база даних
- `redis` — кеш
- `minio` — файлове сховище
- `minio-init` — ініціалізація bucket

## ⚙️ Запуск проєкту

### 1. Клонування

```bash
git clone <repo>
cd project
```

### 2. Налаштування .env

Створіть файл `.env` у корені проєкту:

```env
PORT=4000


DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=events
DB_PORT=5432

MINIO_USER=minio
MINIO_PASSWORD=minio123
```

### 3.Запуск

```
docker-compose up --build
```
